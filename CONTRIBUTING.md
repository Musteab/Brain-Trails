# Contributing to Brain Trails 🗡️📜

First off, thank you for considering contributing to Brain Trails! It's people like you that make this study companion truly legendary. 

Whether you're squashing bugs, adding new cosmetics, or refining the AI Familiar, every contribution helps our users level up their study sessions.

## 🗺️ The Developer's Quest Log

### 1. Find a Quest (Issue)
* Check out the [Issues](https://github.com/Musteab/Brain-Trails/issues) tab. 
* Look for issues tagged with `good first issue` or `help wanted` if you're new to the codebase.
* If you want to work on something, just drop a comment saying "I'd like to take this on!" so we don't duplicate efforts.

### 2. Gear Up (Local Setup)
Make sure you have Node.js 20+, Python 3.12+, and a Supabase project ready. 
1. Fork the repo and clone your fork.
2. Run `npm install` for the frontend.
3. Set up your `.env.local` (see the README for keys).
4. (Optional) Set up the Python virtual environment for the AI backend.
5. Create a new branch for your feature: `git checkout -b feature/your-awesome-feature`.

### 3. Slaying the Code (Development)
* **Keep it clean:** We use Next.js App Router and Tailwind CSS. Try to match the existing "Nintendo meets Notion" design philosophy.
* **Test your spells:** Make sure the app builds successfully (`npm run build`) and doesn't introduce linting errors (`npm run lint`).

### 4. Submit for Review (Pull Request)
* Push your branch to your fork.
* Open a Pull Request (PR) against the `main` branch of the original repository.
* Fill out the PR template completely. Include screenshots if you changed the UI!

## 🤝 Code of Conduct
By participating in this project, you agree to maintain a welcoming, inclusive, and harassment-free environment for everyone. Be kind to your fellow guild members!

## 📬 Need Help?
If you're stuck on a setup step or need architectural advice, feel free to open a Discussion or reach out to Mustafe Abdilahi Ahmed (Maintainer).
